# 学习材料多角度适合度审计

对任何学习材料（教材章节、自编笔记、视频课程大纲、学习路线、冲刺计划）进行系统化的多角度适合度评估，并产出分层优化方案。

## 核心原则

1. **证据驱动，不凭感觉**。每个判断必须引用材料中的具体文本作为证据。
2. **从学习者画像出发**。不是"这个材料好不好"——是"这个材料对理解型学习者好不好"。
3. **给出可操作方案**。不只是打勾打叉，要给出改动成本明确的分层优化建议。

## 五维度评估框架

每个维度对照理解型学习者的核心需求（框架先行、深度加工、主动建构、因果逻辑、可消化节奏），逐条打分。

评分符号：
- ✅ = 满足（材料明确设计了这个）
- ⚠️ = 勉强（材料隐含有但没显式标注）
- ❌ = 缺失（材料完全没有这个设计）

### 角度1：框架先行

| 原则 | 检查点 |
|---|---|
| 学习前搭建框架 | 有没有"这章解决什么问题"？有没有"和前后章什么关系"？ |
| 看目录/思维导图 | 概念层级之间有箭头/关系说明吗？ |
| 带着问题预习 | 有没有预习引导问题？章首有没有核心问题列表？ |

**典型反面证据**：直接开始下定义，不说"为什么需要这个概念？没有它之前怎么处理的？"

### 角度2：深度加工

| 原则 | 检查点 |
|---|---|
| 推导公式来源 | 有推导是"展示推导"（passive）还是"引导推导"（active）？ |
| 追问原理 | 有没有"为什么""如果...会怎样"的追问？ |
| 主动关联旧知 | 有没有"回忆第X章"或"这和XX知识类似"的跨章链接？ |
| 区分本质与表象 | 多个例子/概念是平铺还是总结本质区别？ |

**典型反面证据**：从严平稳的联合分布定义出发推到弱平稳——"从抽象到具体"，理解型学习者更适应"从具体到抽象"。

**关键判断**：材料是从具体直觉走向抽象定义，还是从抽象定义往下推？前者✅，后者⚠️/❌。

### 角度3：主动建构

| 原则 | 检查点 |
|---|---|
| 费曼自测/自问自答 | 有没有"用自己的话讲一遍"的提示？ |
| 停顿思考点 | 有没有"这里停一下，你想想"的显式停顿？ |
| 自查清单 | 有没有"检查自己是否理解了"的清单？ |

**典型反面证据**：连续叙述，无任何停顿、无任何自查提示、无任何"你试试"引导。

### 角度4：因果逻辑链

| 原则 | 检查点 |
|---|---|
| 因果/递进/对比显式标注 | "因为...所以..."链条是否显式写出？ |
| "推导目的是什么"说明 | 推导完成后有没有总结"这个推导告诉我们什么"？ |
| 结论的"所以呢" | 每个结论有没有"因此..."的应用指向？ |

**典型反面证据**：推导步骤正确但从不总结"注意：展示这个推导的目的是让你看到——XX恰好是YY的标志"。读者要自己领悟连接。

### 角度5：信息密度与节奏

| 原则 | 检查点 |
|---|---|
| 可消化节奏 | 有分段、有总结、有"回头看"吗？ |
| 公式与文字平衡 | 公式块之间有过渡文字吗？有没有"这段在说什么"的翻译？ |
| 章节容量 | 总行数/估计阅读时间是否在单次学习可承受范围？ |

## 证据收集方法

对每个角度，必须：
1. **引用原文** — 至少给一个具体文本片段（行号或引用）
2. **标注典型性** — 这一个例子代表整章的模式，还是偶然现象
3. **避免以偏概全** — 如果材料某处有自查提示但别处没有，标注为"偶有但非系统"

## 总评表格式

评估完成后，产出总评维度表：

| 维度 | 评分 | 说明 |
|---|---|---|
| 内容正确性 | ✅✅✅ | 公式/定义有无错误 |
| 结构层级 | ✅/⚠️/❌ | 标题层级、导航是否清晰 |
| 框架感 | ✅/⚠️/❌ | 有导航但有没有"为什么" |
| 推导深度 | ✅/⚠️/❌ | 推导是否完整，是展示还是引导 |
| 理解型友好度 | ✅/⚠️/❌ | 互动性、因果标注、自查引导 |
| 信息节奏 | ✅/⚠️/❌ | 密度是否可消化 |

**结论**：一句话总结材料适合什么角色（权威参考？自主阅读材料？预习入口？），以及不适合什么角色。

## 优化方案：教学重构

对任何评估结果为"不适合直接使用"的材料，产出**教学重构版**——保证原文信息不失真，按理解型学习路径重组结构和表述。



### 两条路径：标注增强 vs 教学重构

不是所有材料都需要体内翻转。根据原文的"抽象→具体"程度，选择不同路径：

| | 标注增强 | 教学重构 |
|---|---|---|
| **定义** | 在外围加 🔑💭✏️🔗 标注，📖 体内段落顺序不动 | 重排 📖 体内段落顺序，翻转"抽象定义→具体推论"为"具体直觉→抽象定义" |
| **适用场景** | 原文概念出场顺序已符合 直觉→抽象（如具体模型先展示再推导） | 原文从抽象定义出发往下推结论（如严平稳→弱平稳） |
| **改动量** | 加标注块，不碰原文段落 | 重排原文段落 + 加标注块 |
| **判断标准** | 正文是否从一个**抽象定义**出发推出子结论？是 → 教学重构；否 → 标注增强 |

**实操判断流程：**

1. 读完一个概念的正文段落
2. 问：第一个出现的正式定义是具体的还是抽象的？
3. 如果抽象（如"严平稳 = 联合分布平移不变"）→ 需要教学重构，翻转顺序
4. 如果具体（如"随机游动 = X_t = X_{t-1} + e_t"）→ 标注增强即可

**反面示例**：平稳性概念——正文以严平稳的联合分布定义开头，然后推出弱平稳条件。这需要教学重构：先展示弱平稳的三个直觉需求，再上升到严平稳作为抽象。

### 原则

1. **信息不失真，可增不可减**：原文所有定义、公式、推导、例子必须保留（可优化表述），还可以新增例子、补充定义、添加公式——只要不删原文信息，加什么都行。唯一标准是最终版通过五维度评价体系。
2. **结构按认知路径重组**：每个概念按"直觉→推导→因果→自查"主线重新组织
3. **以五维度为优化目标**：
   - 框架先行 → 加章首核心问题 + 前后章关系
   - 深度加工 → 从具体直觉出发走向抽象定义（反转"定义→推导"顺序）
   - 主动建构 → 嵌入停顿点（🤔 停一下）、自查问题、费曼提示；每个概念前加 💭 引导思考（场景式提问）
   - 因果逻辑 → 每条推导链显式标注 💡 因果链
   - 信息节奏 → 分段 + 段落小结 + 章尾回顾

### 内容结构规范

教学重构版中，每个核心概念按以下结构组织。**🔑 和 💭 紧接在概念标题之下、正文之上——每个概念有自己的场景引导入口，不是章首的独立前置块。**

```
## {概念标题}
🔑 核心问题（一句话——这个概念回答什么问题）

💭 引导思考（场景式提问，先不查书，用直觉回答）
  场景A：[日常场景] —— 指向概念的身份/本质
  场景B：[关联场景] —— 指向概念之间的关系
  场景C：[极端/对比场景] —— 指向概念的边界或后果

📖 正文（从直觉建立到形式化定义再到推导验证）
  ├── 直觉建立：从场景ABC出发，引出形式化需求
  ├── 形式定义：数学定义、公式
  ├── 推导链路：💡 因果链标注
  └── 回到场景：用刚学会的概念重新解释场景ABC

🤔 自查（合上书，用自己的话讲一遍；讲不通 = 没真懂）

🔗 前后链接（前置知识 ← 本概念 → 后续用途）
```
#### 📖 体内翻转：何时反、怎么反

⚠️ **标注解决不了体内顺序问题。** 如果正文是"抽象定义→具体推论"，只加外部标注（🔑💭✏️🔗）是不够的——读者进入正文立刻被抛回抽象起点。

**判断是否需要翻转：** 读正文第一个完整段落。如果它从以下模式开始 → 需要翻转：
- "如果对一切时滞 k 和时点...都有联合分布相同，则称过程为严平稳的"（抽象定义先行）
- "设 X 为一个...定义为..."（无直觉铺垫的定义先行）

**翻转模板：**

原顺序：抽象定义 → 公式推导 → 具体推论 → 弱化版本定义
新顺序：
1. 直觉建立（从场景 ABC 引出"三个直觉需求"）
2. 形式定义（基于直觉需求给出**具体版本**的定义——如弱平稳三条件）
3. 推导链路（💡 因果链标注——展示如何从具体定义上升到抽象定义）
4. 回到场景（用刚学的概念重新解释场景）

**不翻转的情况：**
- 具体模型的推导（随机游动：先给公式，再算均值/方差/协方差——合理）
- 基础定义型概念（"随机过程 = 随机变量序列"——无抽象阶梯）



#### 引导思考的场景设计

**三个场景的职能：**

| 场景 | 职能 | 设计方法 |
|---|---|---|
| A | 身份/本质 | "你觉得哪个更..."——迫使学习者区分本质与表象 |
| B | 关系/关联 | "有相关性吗？远和近有区别吗？"——暗示距离衰减 |
| C | 极端/后果 | "如果...100天后会怎样？"——暴露非平稳特性 |

**设计原则：**

1. **用日常经验，不用数学**。不说"均值和方差"，说"平均气温和气温波动程度"。
2. **不给答案，给方向**。问题本身暗示正确思考方向，但结论留给学习者自己得出。
3. **每个概念配 2-3 个场景**，核心概念必配 3 个（A+B+C），辅助概念可配 2 个。
4. **场景 A 先于 B，B 先于 C**——从简单识别到复杂推理，梯度递进。
5. **场景可跨概念复用**——同一个场景（如气温记录）可以串联多个概念，形成叙事连贯性。

**示例（平稳性概念）：**

```
💭 引导思考（先不查书，用直觉回答）

场景A：你记录每天的气温。直觉上，"平均气温"和"气温波动程度"哪个
更能概括这条序列的整体特征？——这就是均值 vs 方差的意义。

场景B：今天的温度和昨天的温度，你觉得有相关性吗？今天和一个月前呢？
——联系越远越弱，这就是自协方差要度量的东西。你觉得怎么用一个数来
表达"今天和昨天有多相关"？

场景C：假如每天温度变化完全随机（涨跌不可预测），过了100天，你对
当前温度的不确定性比第1天大还是小？为什么？——这指向随机游动的方差
特性。
```

#### 标注插入速查表

| 标注 | 插入位置 | 常见错误 |
|---|---|---|
| 🔑💭 | 节标题**下方**第一行，正文之前 | 放在标题前面 → 游离在节外 |
| 💡 因果链 | 推导段落**之后**，下一话题之前（段落间，不嵌句子内） | 嵌在句子中间拦腰截断 |
| ✏️ 自查 | 概念块**末尾**（📖 正文完成后），自成一段 | 答案紧跟问题 → 退化为修辞反问 |
| 🔗 链接 | 概念块末尾，与 ✏️ 并列 | — |
| 📦 回顾 | 章尾、`## 小结` 之前 | — |

**Python 处理 LaTeX 警告：** `\u000b` `\f` `\u0007` `\b` 是 Python 转义字符。处理含 `\varepsilon` `\frac` `\beta` 的文本时，用原始字节（`b'\v'`）或 raw string，避免控制字符污染。


### 工作流

1. 五维度评估原文 → 产出基线评分表 + 问题清单
2. 按评估结果直接生成教学重构版
3. 呈现给用户，询问是否修改

### 输出

评估报告包含：
- 原文五维度评分表
- 完整教学重构版
- 改动说明（结构变化 / 新增内容 / 保留内容）

## 适用对象

本 skill 适用于评估：
- 教材章节/整本教材
- 自编学习笔记或他人笔记
- 视频课程大纲/syllabus
- 学习路线/冲刺计划
- 任何"我要用这个东西来学习"的材料

不适用于：
- 评估考试题目（那是另一个维度——难度、区分度、覆盖度）
- 评估 AI 生成的直觉短文（那些本身就应该已经是理解型友好的）
- OCR 修复质量评估（那是 comprehension-study-design 的 OCR 修复流程范畴）

## 输出格式

一次完整的审计报告包含：

```
# {材料名}适合度评估与多角度优化方案

## 一、多角度适合度评估
### 角度1-5（逐一展开，配证据表+正文示例）

## 二、总评
### 总评维度表 + 结论（一句话）

## 三、优化方案
### 教学重构版（完整内容）

### 改动说明 + 待用户确认（可修改）
```

## 常见陷阱

1. **不用感觉判断**。❌"感觉这篇材料读起来不友好"；✅"材料在 L45 直接给出定义，没有前置问题——不符合角度1的框架先行原则"。
2. **不只打勾不打方案**。评估完必须给优化方案，否则是半成品。
3. **不越界**。如果材料的问题是 OCR 破损/公式错误，不要用本 skill 的框架去评估"适合度"——先告诉用户"内容本身有错误，需要先修复再评估"。OCR 修复流程见 `comprehension-study-design` skill 的 `references/textbook-ocr-repair.md`。
4. **不假设用户的学习状态**。评估的是材料本身对理解型学习者的适合度，不是"用户当前能不能读懂"。
5. **优化方案不等于终稿**。教学重构版产出后必须交给用户确认，用户可能要求修改——这不是否定方案质量，是正常协作流程。
6. **教学重构版生成**。本 skill 定义评估标准，具体的标注插入方法和实现陷阱见：
   - 评估流程和迭代闭环 → 本 skill 正文
   - 标注插入规范（💡✏️🔗📦）→ `references/guided-thinking-template.md`
   - **实现陷阱**（Python 转义字符导致 LaTeX 破损、标注块插入位置错误、R 代码格式化等）→ `references/textbook-pedagogical-pitfalls.md`

## 插入标注的执行陷阱

这些是生成教学版时的具体技术坑，不是评估维度的哲学问题。

### 🔑💭 放置位置：标题内 vs 标题外

❌ **错误**：🔑💭 块放在 `## 章节标题` **之前**——产生游离在章节外的"前置块"。

✅ **正确**：🔑💭 块放在标题 **下面、正文第一行之前**——成为该章节的内容入口。

```
## 2.3平稳性

🔑 **核心问题**：...
💭 **引导思考**...
[正文开始]
```

### 💡✏️ 放置位置：段落之间 vs 句子中间

❌ **错误**：💡✏️ 块插在一句话的中间，把句子拦腰截断（如"相关系数是...的度量 > 💡 因果链... > ✏️ 请动笔... ，而某种程度上..."）。

✅ **正确**：💡 紧接推导段落 **之后**（段落间），✏️ 在概念块末尾（进入下一概念前）。两者都是独立的段落级标注，不嵌入任何句子内部。

### LaTeX `\varepsilon` 的 Python 陷阱

在 Python 字符串中写入 `\varepsilon` 时，`\v` 是垂直制表符（VT, 0x0B）的转义序列。文件中的 `\varepsilon` 会变成 `<VT>arepsilon`，公式无法渲染。

**修复**：在 Python bytes 操作中，用十六进制绕过转义：

```python
correct = b'\x5c\x76arepsilon'   # 0x5C=\ 0x76=v → \varepsilon
broken  = b'\x0barepsilon'       # 0x0B=VT → 被破坏的序列
raw = raw.replace(broken, correct)
```

7. **🤔 自查 ≠ ✏️ 建构练习**。教学重构版中插入的"自查"节点容易退化为修辞性反问——提问后立刻给出答案，读者无需真正停下来推导或动笔。正确的做法：每个自查节点必须要求具体动作（"请动笔写出""试构造""动手算""列表对比"）且答案不紧跟在问题后面。🤔 格式若只有修辞提问无动作要求 → 升级为 ✏️ 格式。
8. **R 代码块格式**。教材原文中的 R 代码常以裸文本或 `>` 引用块形式出现（如 `> plot(rwalk,...)`）。教学版必须将其转为 ` ``` ```r ``` ` 代码块并加一行说明（"考试不要求写代码，仅作参考"）。
9. **抽象→具体 翻转判断**。仅当概念本身是"从抽象定义往下推"（如严平稳→弱平稳）时才翻转。具体模型的推导（如随机游动、滑动平均）不需要翻转——模型先展示再推导是合理的。判断标准：正文是从一个抽象定义出发推出子结论 → 翻转；正文是从具体公式推导性质 → 不动。

## 插入标注的执行陷阱

这些是生成教学版时的具体技术坑，不是评估维度的哲学问题。

### 🔑💭 放置位置：标题内 vs 标题外

❌ **错误**：🔑💭 块放在 `## 章节标题` **之前**——产生游离在章节外的"前置块"。

✅ **正确**：🔑💭 块放在标题 **下面、正文第一行之前**——成为该章节的内容入口。

```
## 2.3平稳性

🔑 **核心问题**：...
💭 **引导思考**...
[正文开始]
```

### 💡✏️ 放置位置：段落之间 vs 句子中间

❌ **错误**：💡✏️ 块插在一句话的中间，把句子拦腰截断（如"相关系数是...的度量 > 💡 因果链... > ✏️ 请动笔... ，而某种程度上..."）。

✅ **正确**：💡 紧接推导段落 **之后**（段落间），✏️ 在概念块末尾（进入下一概念前）。两者都是独立的段落级标注，不嵌入任何句子内部。

### LaTeX `\varepsilon` 的 Python 陷阱

在 Python 字符串中写入 `\varepsilon` 时，`\v` 是垂直制表符（VT, 0x0B）的转义序列。文件中的 `\varepsilon` 会变成 `<VT>arepsilon`，公式无法渲染。

**修复**：在 Python bytes 操作中，用十六进制绕过转义：

```python
correct = b'\x5c\x76arepsilon'   # 0x5C=\ 0x76=v → \varepsilon
broken  = b'\x0barepsilon'       # 0x0B=VT → 被破坏的序列
raw = raw.replace(broken, correct)
```

### 操作 .md 文件的铁律

**编辑含 LaTeX 的 .md 文件时，永远不要用 Python（execute_code）。** Python 的 `\n` `\r` `\v` `\f` `\b` `\a` 会立即破坏 `\nabla`、`\rho`、`\frac`、`\varepsilon`、`\begin`。

一律用 Hermes 原生工具：
- `patch` —— 替换文本块
- `write_file` —— 全量写
- `terminal` + `sed` —— 批量替换
