# requirement-validate Skill

需求理解确认闭环。调用此 skill 后，Claude 用自己的话复述对需求的完整理解，用户审查并修改，直到确认无误。

**只做需求确认，不进入实现阶段。**

## 用法

用户带上需求文本调用此 skill：

> @requirement-validate 帮我做一个用户登录功能...

Claude 会复述理解 → 用户确认/修正 → 循环直到确认。

## 触发场景

- "需求确认"
- "帮我理解需求"
- "验证需求"
- "requirement check"
- "确认一下需求"
- "先别写代码，先确认"
- 任何要求先确认再动手的场景

## 不适用

- 纯技术问答
- 代码审查
- bug 修复（不需要需求梳理的场景）

## 原理

强制一个轻量级反馈循环：

```
用户说需求 → Claude 复述 → 用户检查 → 对了结束 / 错了修正 → 重新复述全部 → 确认
```

每次修改后重新复述**全部**需求，确保累积理解不遗漏。
