# vibe-learn

让非专业人士用专业语言向 AI 描述需求。不学底层原理，只学怎么把话说清楚。

## 为什么需要这个

你脑子里知道想做什么，但说出来是"帮我做个好看的网页"、"帮我搭个后端"——AI 听不懂。

AI 听不懂的原因不是你缺少开发知识，而是你没有用专业语言描述你的需求。

这个 skill 做的事：每个话题告诉你**这是什么、怎么用、怎么跟 AI 说清楚**。不讲算法、不讲数学、不讲底层原理。只讲够用的。

## 核心理念

> AI 时代，你不需要成为工程师。你需要的是知道怎么让 AI 理解你。

- 不教原理，只教怎么用
- 不讲底层，只给你需要的术语
- 每次教学附带 Prompt 模板，复制就能用

## 安装

### 方式一：直接下载（推荐）

```bash
# 1. 进入 Claude Code 的 skills 目录
cd ~/.claude/skills

# 2. 克隆这个仓库
git clone https://github.com/HP-Patience/vibe-learn.git

# 3. 重启 Claude Code
```

### 方式二：手动复制

```bash
# 1. 下载本仓库的 ZIP 并解压
# 2. 将解压后的 vibe-learn/ 文件夹整个复制到 ~/.claude/skills/ 下
# 3. 重启 Claude Code

# 最终路径应该是：
# ~/.claude/skills/vibe-learn/SKILL.md
# ~/.claude/skills/vibe-learn/index.md
# ~/.claude/skills/vibe-learn/trees/...
```

### 方式三：从 release 下载

到本仓库的 Releases 页面下载最新版 `vibe-learn.zip`，解压到 `~/.claude/skills/` 下。

### 验证安装成功

在 Claude Code 中输入 `/vibe-learn` 并回车，看到教学提示即为安装成功。

## 使用案例

### 案例一：不知道怎么跟 AI 描述后端需求

> 用户输入：`/vibe-learn 我想搭建一个用户登录注册的后端`

AI 教你：后端是什么、RESTful API 怎么设计、JWT 认证流程、怎么跟 AI 说"帮我写一个登录接口"。学完你就能这样跟 AI 说：

> "帮我用 FastAPI 写一个登录接口，用 JWT 做认证，POST /api/auth/login，接收 email 和 password，返回 access_token。校验邮箱格式、密码至少6位、失败返回 401。"

### 案例二：想学 Git 但不知道从哪开始

> 用户输入：`/vibe-learn 我想学 Git 分支管理`

AI 教你：什么是分支、什么时候创建分支、merge 和 rebase 的区别、冲突怎么解决。学完你就能这样跟 AI 说：

> "我在 feature/payment 分支开发完了，帮我合并到 main 分支。先拉取最新的 main，然后用 merge 方式合并，如果有冲突告诉我怎么解决。"

### 案例三：想做产品但不会写需求

> 用户输入：`/vibe-learn 产品经理怎么分析需求`

AI 教你：用户故事怎么写、MVP 怎么做、功能优先级怎么排（MoSCoW 法则）。学完你就能这样跟 AI 说：

> "我要做一个给自由职业者用的记账工具。目标用户是不懂财务的独立设计师。MVP 功能：拍照识别发票、自动分类收支、年底导出报税报表。帮我按 MoSCoW 排优先级，先做 Must Have。"

### 案例四：想把项目部署上线

> 用户输入：`/vibe-learn 怎么把项目部署到线上`

AI 教你：部署三步骤（Build/Serve/Deploy）、Vercel 怎么用、域名 HTTPS 怎么配、CI/CD 是什么。学完你就能这样跟 AI 说：

> "帮我把这个 Next.js 项目部署到 Vercel。需要配置环境变量 DATABASE_URL 和 JWT_SECRET，绑定我的域名 example.com，开启 HTTPS 自动续期。"

---

**核心逻辑**：学完一个技能点 → 你就能用专业术语把想法翻译成 AI 能懂的 Prompt。

## 每次你会得到

| 模块 | 解决什么问题 |

| 一句话 | 这东西是干什么的，非科班能懂 |

| 常见坑 | 新手最容易掉进去的，提前避开 |
