# Vibe Learn — 为Vibe Coder准备的技能教学

## 核心原则

1. **只教"怎么用"，不教"为什么这样设计"**
2. **每项技能必须回答三个问题：**
   - 这是什么（一句话概念，非科班能懂）
   - 怎么用它（实操步骤 + AI Prompt模板）
   - 怎么向AI描述需求（专业术语速查 + Prompt套路）
3. **不讲底层原理、不讲数学推导**
4. **每个知识点必须有"向AI描述需求的Prompt框架"**

## 工作流

当用户调用此skill并说明想学什么时：

1. 读取 `index.md` → 根据用户问题匹配关键词 → 定位对应的技能树文件
2. 读取匹配到的 `trees/` 文件 → 找到具体技能点
3. 按下面的**教学输出模板**生成教学内容
4. 如果用户问题匹配到多个技能点，先列出建议清单让用户选择

### 知识缺失处理

当用户提问的知识在技能树中 **找不到匹配** 时：

#### Step A：判断归属

检查该知识是否属于现有19个技能树领域之一。

**属于现有领域** → 使用 AskUserQuestion：

> "`[知识名称]` 不在当前技能树中，但它属于 `[领域名称]` 领域。要加入吗？"
>
> A) 加入 `trees/[编号]-[领域].md` — 追加为新技能点
> B) 不加入，仅本次教学

选A → 按教学模板输出内容 → 追加到对应 trees 文件末尾（格式：`- **[技能名]**: [一句话描述]。关键词: kw1, kw2, kw3`）→ 更新 index.md 关键词列表。

**不属于任何现有领域** → 使用 AskUserQuestion：

> "`[知识名称]` 不属于现有19个领域。要新建技能树领域吗？建议名称："
>
> A) `[候选名称1]` — [一句话描述]
> B) `[候选名称2]` — [一句话描述]
> C) `[候选名称3]` — [一句话描述]
> D) 不新建，仅本次教学

选A/B/C → 创建 `trees/[下一个编号]-[slug].md` → 加入该技能点 → 更新 index.md 索引表。

#### Step B：领域命名原则

- 尽最大努力与其他19个领域独立，避免概念重叠
- 名称简短（2-4字），能一眼看出是干什么的
- 候选3个名称应覆盖不同角度（如：按角色命名 vs 按活动命名 vs 按产物命名）

## 教学输出模板

每次教学必须按此模板输出：

```
# [技能点名称]

## 一句话
[用非科班能懂的语言，一句话说清楚这个是什么]

## 你什么时候需要它
[2-3个典型场景，让用户知道"哦我现在就需要这个"]

## 怎么用
[具体操作步骤，每步配一个AI Prompt模板]
→ AI Prompt：`[直接可复制给AI的提示语]`

## 专业表达速查
| 术语 | 通俗含义 |
|------|----------|
| xxx  | yyy     |

## 怎么向AI描述需求
**Prompt框架：**
`[描述背景] + [说清楚目标] + [指定技术栈] + [给出约束条件]`

**实操示例（2-3个）：**
1. 简单场景：`[Prompt示例]`
2. 复杂场景：`[Prompt示例]`
3. 排错场景：`[Prompt示例]`

## 常见坑
- 坑1：[描述 + 为什么会掉进去 + 怎么避开]
- 坑2：...
```

## 索引结构

`index.md` 维护19个技能树文件的：
- 文件名
- 一句话描述
- 关键词列表（用于匹配用户问题）
- 包含的技能点列表

当用户问题无法精确匹配时，用 `index.md` 的完整技能点列表帮用户找到最接近的话题。
