# 09 — 测试

> 写代码验证代码是否正确，改代码时不怕改坏

## 技能点列表

- **为什么要测试**：手动测每次改代码都要重测所有功能→烦→不测→上线崩→跑路。测试写一次跑一辈子。关键词: 测试, 为什么测试, 测试价值
- **测试金字塔**：底层单元测试(多快)→中层集成测试→顶层E2E(少慢)。写越多单元测试越好。关键词: 测试金字塔, 单元测试, 集成测试, E2E
- **单元测试**：测试单个函数/组件。给输入→检查输出。不依赖数据库/网络。关键词: 单元测试, unit test, 函数测试
- **集成测试**：测试多个模块一起工作。比如"用户注册→发邮件→数据库有这条记录"。关键词: 集成测试, integration test
- **E2E(端到端)测试**：模拟真实用户点击操作整个流程。打开浏览器→登录→点按钮→看结果。Playwright/Cypress。关键词: E2E, 端到端, Playwright, Cypress
- **测试框架速览**：Jest(JS/TS通用)、Vitest(Vite生态、快)、Pytest(Python)、Playwright(浏览器E2E)。关键词: Jest, Vitest, Pytest, Playwright
- **断言(Assertion)**：检查是否符合预期的语句。`expect(add(2,3)).toBe(5)`——如果不是5测试失败。关键词: assert, expect, 断言
- **Mock/Stub**：模拟外部依赖。测发邮件功能时不真发邮件，模拟sendEmail函数返回值。关键词: mock, stub, 模拟, 打桩
- **测试覆盖率**：代码有多少被测试执行过。80%算不错，不追求100%。关键词: 覆盖率, coverage, 测试覆盖
- **TDD(测试驱动开发)**：先写测试→测试红(失败)→写代码→测试绿(通过)→重构。关键词: TDD, 测试驱动, red-green-refactor
- **BDD(行为驱动开发)**：用自然语言描述行为→自动生成测试。Given/When/Then格式。关键词: BDD, 行为驱动, Given, When, Then
- **怎么让AI帮你写测试**：把要测的函数/组件给AI→告诉AI框架(Jest/Pytest)→告诉AI想要什么边界情况。AI写测试很擅长。关键词: AI写测试, prompt, 生成测试
- **好测试的标准**：可读(看名字知道测什么)、独立(不依赖其他测试)、快(秒级)。关键词: 好测试, 测试原则
- **CI中跑测试**：GitHub Actions里配好，每次PR自动跑测试，不通过不允许合并。关键词: CI, 自动测试, 流水线
