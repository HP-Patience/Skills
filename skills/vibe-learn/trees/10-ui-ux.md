# 10 — UI/UX设计

> 设计好看好用的界面，理解设计师在说什么

## 技能点列表

- **UI和UX区别**：UI=界面长什么样(颜色/字体/间距)；UX=用起来什么感受(流程顺不顺/找不找得到按钮)。关键词: UI, UX, 界面, 体验, 区别
- **设计系统(Design System)**：一套统一的设计规则和组件库。比如按钮颜色、字体大小、间距全统一，不各自发挥。关键词: design system, 设计系统, 组件库
- **Design Tokens**：把设计值(颜色/字体/间距)抽象成变量，一处改全局变。`--primary: #3B82F6`。关键词: design tokens, css variables, 设计变量
- **组件库怎么用**：shadcn/ui(复制粘贴、可定制)、Ant Design(企业级开箱即用)、MUI(Google风格React组件)。关键词: shadcn, ant design, MUI, 组件库
- **响应式 vs 自适应**：响应式=一个设计自动适配所有屏幕；自适应=为手机/平板/电脑各设计一套。现在都用响应式。关键词: responsive, 响应式, 自适应
- **移动优先(Mobile First)**：先设计手机版，再扩展到桌面。手机屏幕小强迫你做取舍。关键词: mobile first, 移动优先
- **常见UI组件模式**：导航栏、表单(输入/选择/提交)、卡片(信息块)、弹窗(Modal)、表格。关键词: 组件, 导航, 表单, 卡片, 弹窗, 表格
- **尼尔森十大可用性原则**：设计权威的10条经验。最重要：系统状态可见(正在加载要有loading)、用户可控(能撤销)、一致性(同一功能别换位置)。关键词: 可用性, 尼尔森, heuristic
- **信息架构(IA)**：页面内容和导航怎么组织。用户能不能3秒内找到想要的信息。关键词: 信息架构, IA, 导航设计
- **用户流程(User Flow)**：用户完成一个任务的完整路径。注册流程：首页→点注册→填表→验证邮箱→成功页。关键词: user flow, 用户流程, 流程图
- **线框图 vs 高保真**：线框=灰白草图(定结构和流程)；高保真=和最终产品一样的精细设计。关键词: wireframe, 线框图, hi-fi, 高保真
- **设计工具概览**：Figma(最主流，在线协作)、即时设计/MasterGo(国内替代)。关键词: Figma, 即时设计, 设计工具
- **如何看懂Figma设计稿**：关注四个东西——布局(东西在哪)、颜色(色值)、字体(字号字重)、间距(padding/margin数值)。关键词: Figma, 设计稿, 标注, 切图
- **如何向AI描述视觉需求**：说清楚风格(极简/科技/可爱)+配色方向(蓝白/黑金)+参考网站+具体组件需求。关键词: 视觉需求, AI生成UI, 描述设计
- **颜色理论直觉**：主色(品牌色，占比最大)、辅助色(次要元素)、中性色(背景/文字灰阶)、语义色(绿成功/红危险/黄警告)。关键词: 颜色, 配色, 主色, 色板
- **字体选择直觉**：衬线体(有装饰，正式/古典)、非衬线(没装饰，现代/干净)、等宽(写代码用)。正文用非衬线。关键词: 字体, font, 衬线, 非衬线
- **间距与对齐(8px网格)**：间距和元素尺寸都用8的倍数(8/16/24/32)。视觉自然对齐。关键词: 间距, spacing, 8px grid, 网格
- **可访问性(a11y)**：让残障用户也能用。核心：颜色对比度够高、所有交互能用键盘、图片有alt文字。关键词: a11y, 无障碍, 对比度, 屏幕阅读器
- **动效与微交互**：按钮hover变色、loading动画、页面切换过渡。提升品质感但别过度。关键词: 动效, 动画, 微交互, transition
- **设计评审怎么参与**：关注三点：这个设计能不能实现(技术可行性)？和现有设计一致吗？用户能看懂怎么用吗？关键词: 设计评审, design review
