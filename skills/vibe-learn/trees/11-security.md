# 安全意识

- **为什么安全不是"以后再说"**: 上线前没考虑安全=给黑客留门。从一开始就养成好习惯，安全是贯穿开发全过程的意识而非事后修补。 关键词: 安全意识、上线前检查、安全习惯、DevSecOps
- **OWASP Top 10是什么**: 业界公认十大Web安全风险榜单，每2-3年更新一次。不用背，知道有这东西、用到时查。包括注入、认证失效、敏感数据泄露等。 关键词: OWASP、Web安全、十大风险、安全标准
- **SQL注入**: 用户输入恶意SQL语句操纵数据库，可能导致数据泄露或删除。防法：永远用参数化查询(prepared statement)或ORM，绝不拼接SQL字符串。 关键词: SQL注入、参数化查询、ORM、输入过滤、数据库安全
- **XSS(跨站脚本)**: 用户输入恶意脚本被其他用户浏览器执行，可窃取cookie/token。防法：输出HTML时转义(escape)特殊字符，用CSP头限制脚本来源。 关键词: XSS、跨站脚本、HTML转义、CSP、输出编码
- **CSRF(跨站请求伪造)**: 诱导已登录用户点击链接或访问页面，以用户身份执行非自愿操作。防法：CSRF token、SameSite Cookie、验证Referer头。 关键词: CSRF、请求伪造、token验证、SameSite、Referer
- **密钥管理**: API密钥、数据库密码、第三方secret全部放.env文件，.env坚决加入.gitignore。代码中硬编码密钥=将密码公开到互联网。 关键词: 密钥管理、.env、.gitignore、环境变量、密钥泄露
- **密码存储**: 永远不存明文密码。用bcrypt/argon2等哈希算法加密，加盐(salt)防彩虹表攻击。哈希不可逆，即使数据库泄露密码也不暴露。 关键词: 密码哈希、bcrypt、argon2、加盐、彩虹表
- **HTTPS强制**: HTTP明文传输可被中间人截获密码和cookie。HTTPS加密通信，用TLS证书。生产环境强制301重定向HTTP到HTTPS。 关键词: HTTPS、TLS、SSL证书、中间人攻击、加密传输
- **权限校验(RBAC)**: Role-Based Access Control，用户→角色→权限三层模型。先定义角色(如admin/editor/viewer)，给角色分配权限，再把用户绑定到角色。 关键词: RBAC、权限控制、角色管理、访问控制、授权
- **CORS安全配置**: 跨域资源共享别偷懒设Access-Control-Allow-Origin: *。明确列出允许的域名，用白名单模式，不信任所有来源。 关键词: CORS、跨域、白名单、Allow-Origin、浏览器安全
- **输入校验**: 永远不信任用户输入。前端校验防用户犯错(体验)，后端校验防黑客攻击(安全)。数据类型、长度、格式、白名单验证全覆盖。 关键词: 输入校验、白名单验证、后端校验、数据清洗、防御性编程
- **依赖安全漏洞**: 第三方包有漏洞你的项目就有漏洞。定期npm audit/pip audit检查，及时更新有CVE的依赖。用Dependabot自动提PR。 关键词: 依赖漏洞、npm audit、CVE、Dependabot、供应链安全
- **限流(Rate Limiting)**: 防止同一IP暴力请求攻击(撞库、刷接口、爬数据)。设限：同一IP每分钟最多N个请求，超出返回429。用express-rate-limit或Nginx限流。 关键词: 限流、Rate Limiting、暴力攻击、429、接口保护
- **安全事件响应**: 确认入侵→立刻封堵漏洞(热修复)→评估数据泄露范围→通知受影响用户→全面修复上线→写复盘文档防止再次发生。 关键词: 安全响应、漏洞修复、数据泄露、复盘、应急响应
