# 项目管理

- **PM(项目经理)是干什么的**: 管进度(什么时候做完)、管风险(什么可能出问题)、管沟通(让所有人知道该做什么)。确保项目按时按质上线，不一定是技术最强的。 关键词: 项目经理、进度管理、风险管理、沟通协调、按时交付
- **敏捷(Agile)**: 快速迭代小步快跑，每1-2周出可用版本，持续收集反馈调整方向。核心：响应变化胜过遵循计划、可用的软件胜过详尽的文档。 关键词: 敏捷开发、迭代、快速反馈、持续交付、Scrum
- **Scrum框架**: Sprint计划会(定这2周做什么)→每日站会(15分钟，昨天做了什么/今天做什么/有什么阻塞)→Sprint回顾(哪些做得好/哪些要改进)。角色：Product Owner管需求、Scrum Master管流程、Team管执行。 关键词: Scrum、Sprint、站会、回顾、PO
- **看板(Kanban)**: 待办(To Do)→进行中(In Progress)→已完成(Done)，可视化工作流。限制同时进行的任务数(WIP限制)，减少上下文切换。比Scrum更轻量，适合运维类工作。 关键词: 看板、Kanban、可视化、WIP限制、工作流
- **任务拆分(WBS)**: Work Breakdown Structure，大任务→中任务→小任务→可执行的子任务。每个子任务不超过1天，可估算、可分配、可验收。拆不细说明你还没想清楚。 关键词: WBS、任务拆分、工作分解、可执行任务、粒度控制
- **估时**: 故事点(相对大小,用Fibonacci数列1/2/3/5/8/13，比较任务之间的相对工作量)→工时(小时/天，绝对时间)→三点估算(最乐观/最可能/最悲观取加权平均)。 关键词: 估时、故事点、工时估算、三点估算、Fibonacci
- **里程碑**: 项目关键节点(MVP上线、测试完成、灰度发布、全量上线)，倒推时间表。里程碑是硬约束，到达时间不可任意推迟，赶不上就砍范围。 关键词: 里程碑、关键节点、倒推计划、硬约束、时间线
- **甘特图**: 横轴时间纵轴任务，每个任务一条横线表示起止时间。一眼看清时间线、任务并行度和依赖关系。不需要画很漂亮，Excel就能做。 关键词: 甘特图、时间线、任务依赖、并行任务、可视化计划
- **技术方案怎么写**: 背景(为什么要做)→方案对比(有哪些方案/各自优缺点)→架构设计(模块划分/数据流)→接口设计(API路径/请求响应格式)→数据库设计(表结构/索引)→排期(里程碑/负责人)。 关键词: 技术方案、架构设计、接口设计、数据库设计、方案对比
- **风险识别**: 什么可能出错？概率多大？影响多大？怎么办(缓解措施+应急预案)？把风险写出来就已经解决了一半。每周过一遍风险清单。 关键词: 风险识别、概率评估、影响分析、缓解措施、应急预案
- **进度同步**: 周报=本周做了什么+下周计划做什么+有什么阻塞需要帮助。简洁明确，不要流水账。阻塞项必须明确"需要谁做什么"，别写"希望能解决"，写"需要张三帮忙查数据库权限"。 关键词: 进度同步、周报、阻塞项、沟通模板、状态更新
- **跨角色协作语言**: 和设计师聊交互细节/响应式/动效规范→和前端聊接口约定/组件状态/错误处理→和后端聊数据模型/接口设计/性能约束→和测试聊用例/边界条件/验收标准。 关键词: 跨角色协作、沟通语言、前后端协作、设计对接、测试配合
- **需求变更怎么处理**: 评估变更影响(工作量/对现有功能的影响/是否影响排期)→给出选项(A延期上线/B砍其他功能/C不接受变更)→让决策者选，不自己扛。 关键词: 需求变更、影响评估、排期博弈、功能取舍、变更管理
- **线上问题处理**: 发现(监控报警/用户反馈)→评估严重性(P0全员上线/P1紧急修复/P2正常排期)→定位责任人→修复上线→验证恢复→写复盘文档(为什么发生/为什么没发现/怎么防止再次发生)。 关键词: 线上问题、故障处理、P0P1P2、复盘、故障分级
- **项目管理工具**: Jira(大团队/复杂流程，功能全但重)→Linear(快且好用，适合中小团队)→Notion(灵活全能，适合文档驱动)→飞书(国内团队，集成沟通和文档)。 关键词: 项目管理工具、Jira、Linear、Notion、飞书
